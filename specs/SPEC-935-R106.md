# SPEC-935-R106 — CI/CD Pipeline + Quality Gates

## Objetivo

Implementar pipeline de Integração Contínua (CI) e Qualidade com:
1. GitHub Actions workflow para testes automáticos em todo PR/push
2. Scripts de qualidade locais (pré-commit, lint, type-check)
3. Gate de cobertura mínima (≥ 80%)
4. Relatório de qualidade consolidado

**Meta:** Elevar maturidade de Automação CI/CD de 6.0 → 8.5/10

## Componentes

### 1. GitHub Actions Workflow (`.github/workflows/ci.yml`)

3 jobs paralelos:
- **lint**: Ruff + mypy + verificação de imports
- **test**: pytest com matriz Python 3.10–3.14, coverage report
- **package**: build dos 3 pacotes pip + teste de importação

### 2. Script de Qualidade Local (`scripts/quality_report.py`)

Gera relatório local com:
- Contagem de testes
- Taxa de aprovação
- Cobertura estimada por módulo
- Erros de lint
- Score de qualidade consolidado

### 3. Quality Gate (`scripts/check_coverage.py`)

Gate que:
- Verifica cobertura ≥ 80%
- Verifica 0 regressões
- Verifica lint sem erros
- Exit code 0 se passa, 1 se falha

## Critérios de Aceitação

1. ✅ `.github/workflows/ci.yml` com 3 jobs (lint, test, package)
