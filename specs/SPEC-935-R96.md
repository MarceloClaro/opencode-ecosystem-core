# SPEC-935-R96: Academic API Gateway

## Objetivo
Criar um gateway REST (FastAPI) que expõe todos os módulos da Universidade Sintética como endpoints HTTP, permitindo integração com ferramentas externas, dashboards e clientes REST.

## Motivação
Atualmente os módulos SU são acessíveis via Python (import) e via MCP (stdio). Falta uma interface REST padrão para integração com sistemas web, CI/CD, e clientes HTTP.

## Critérios de Aceitação (SDD Gate)

### CA1 — Estrutura do módulo
- [ ] Arquivo `synthetic_university/api_gateway.py` define a aplicação FastAPI
- [ ] A aplicação é exposta como `app: FastAPI` (variável global)
- [ ] Se FastAPI não estiver instalado, o módulo carrega sem erro (import condicional)

### CA2 — Endpoints básicos
- [ ] `GET /health` → `{"status": "ok", "version": "1.0.0"}`
- [ ] `GET /tools` → lista de ferramentas disponíveis com nome e descrição

### CA3 — Endpoints de ferramentas
- [ ] `POST /tools/generate` → gera teses (wrap do handler MCP)
- [ ] `POST /tools/evaluate` → avalia tese
- [ ] `POST /tools/enrich` → enriquece tese
- [ ] `POST /tools/visual-abstract` → gera abstract visual
- [ ] `POST /tools/peer-review` → executa revisão
- [ ] `POST /tools/submission` → gera pacote de submissão
- [ ] `POST /tools/novelty` → analisa novidade
- [ ] `POST /tools/dashboard` → gera dashboard

### CA4 — Integração com módulos SU
- [ ] Cada endpoint POST delega para o handler correspondente do MCP server
- [ ] Reusa `handle_generate`, `handle_evaluate`, etc. de `mcp_server.py`

### CA5 — Tratamento de erros
- [ ] Erros retornam `{"error": mensagem}` com status HTTP apropriado (400, 404, 500)
- [ ] Endpoint inexistente retorna 404

### CA6 — Testes
- [ ] Mínimo 8 testes TDD em `tests/test_r96_api_gateway.py`
- [ ] Testes usam `TestClient` do Starlette (sem FastAPI runtime)
- [ ] Todos os testes passam sem rede externa

## Dependências
- `fastapi` (opcional — import condicional)
- `uvicorn` (opcional — para execução)
- `synthetic_university.mcp_server` (handlers)

## Não Escopo
- R96 **não** implementa autenticação/autorização
- R96 **não** implementa rate limiting
- R96 **não** implementa deploy em produção
