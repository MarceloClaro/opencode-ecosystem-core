# SPEC-935-R104c — Compatibility Analysis Document

## Objetivo

Produzir um documento de análise de compatibilidade entre o OpenCode Ecosystem Core
(R82–R103) e o ecossistema externo `OpenCode_Ecosystem-main` (fork de
`timpara/opencode-academic-research`), detalhando gaps, sobreposições e
oportunidades de integração.

## Conteúdo do Documento

O documento (`docs/COMPATIBILITY_ANALYSIS.md`) deve conter:

### 1. Visão Geral dos Ecossistemas

| Aspecto | OpenCode Ecosystem Core | OpenCode_Ecosystem-main |
|---------|------------------------|------------------------|
| Foco | Inteligência acadêmica agentiva | Pipeline de pesquisa acadêmica |
| Skills | 11+ MCP tools | 8 skills OpenCode |
| Agentes | 4+ agentes científicos | 38 agentes MASWOS |
| Cobertura | Descoberta→Evolução→Revisão | Digestão→Survey→Escrita→Revisão→Submissão |

### 2. Mapeamento Funcional

Tabela que mapeia cada skill/comando do fork para módulos equivalentes ou
complementares no nosso core.

### 3. Análise de Gaps

O que cada ecossistema tem que o outro não tem:

**Nosso core tem que o fork não tem:**
- Evolução darwiniana de hipóteses (R101)
- Evidence Graph com busca hierárquica (R102)
- Rubrica 8-dimensões + auditagem em grafo (R103)
- Segurança MCP (R100)

**O fork tem que nosso core não tem:**
- Pipeline completo digestão→survey→escrita→submissão
- 38 agentes especializados (MASWOS)
- Slash commands integrados ao OpenCode
- Template de contribuição com badges

### 4. Plano de Integração

Passo a passo para integrar nossos módulos no fork:

1. Copiar skills de integração (R104a) para o workspace do fork
2. Instalar pacotes pip (R104b) ou copiar módulos
3. Adaptar comandos para usar namespace do fork
4. Registrar MCP tools

### 5. Roadmap de Convergência

Sugestão de evolução conjunta dos dois ecossistemas.

## Critérios de Aceitação

1. ✅ Documento em `docs/COMPATIBILITY_ANALYSIS.md`
2. ✅ Mínimo 5 seções com análise substantiva
3. ✅ Tabelas comparativas com mapeamento funcional
4. ✅ Plano de integração prático e acionável
5. ✅ Testes TDD ≥ 3 (verificam existência e conteúdo do documento)
6. ✅ Registro no EvolutionRegistry
