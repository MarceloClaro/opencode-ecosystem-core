# SPEC-935-R99: Scientific RAG Evolved — Adaptive Retrieval + Outline Synthesis + Citation Graph

## Objetivo
Evoluir o RAG científico existente (SPEC-919) com 3 capacidades avançadas: **retrieval adaptativo** que escolhe estratégia por complexidade da consulta, **grafo de citação** para navegação entre documentos, e **síntese guiada por outline** que planeja, recupera e refina respostas estruturadas.

## Motivação
O `ScientificRAG` atual (SPEC-919) é funcional mas usa estratégia única de retrieval (busca híbrida flat). Consultas complexas — que exigem múltiplas fontes, síntese cross-documento ou raciocínio multi-etapa — são mal servidas por uma única rodada de busca.

## Arquitetura

```
RAGEvolved
│
├── 1. AdaptiveRetriever
│     Análise de complexidade → escolha de estratégia
│     ├── sequential: busca única, retorna top-k chunks consecutivos
│     ├── parallel: múltiplas sub-queries, funde resultados
│     └── recursive: busca → evidência → nova consulta → refinamento
│
├── 2. CitationGraph
│     ├── add_edge(citing, cited)
│     ├── get_related(doc_id) → vizinhos no grafo
│     └── expand_retrieval(evidence) → citações citadas/citantes
│
├── 3. OutlineSynthesizer
│     ├── plan(query) → outline com seções
│     ├── retrieve_per_section(outline) → evidências por seção
│     └── synthesize(outline, evidence) → resposta estruturada
│
└── 4. RAGEvolved (orquestrador)
      analyze_complexity → retrieve adaptativo → expandir citações → sintetizar
```

## Componentes

### C1 — AdaptiveRetriever
Analisa a consulta e seleciona estratégia de retrieval.

| Método | Descrição |
|--------|-----------|
| `analyze_complexity(query)` → `str` | Retorna 'simple', 'moderate', ou 'complex' |
| `retrieve_adaptive(query, rag)` → `List[RetrievedEvidence]` | Roteia para estratégia adequada |
| `_retrieve_sequential(query, rag, top_k)` → `List` | Busca única padrão |
| `_retrieve_parallel(query, rag, top_k)` → `List` | Decompõe em sub-queries, funde |
| `_retrieve_recursive(query, rag, top_k, depth)` → `List` | Busca iterativa com refinamento |

Critérios de complexidade:
- **simple**: ≤ 8 palavras, 1-2 conceitos, pergunta factual
- **moderate**: 8-20 palavras, 2-4 conceitos, pergunta comparativa
- **complex**: > 20 palavras, 4+ conceitos, pergunta multi-etapa ou síntese

### C2 — CitationGraph
Grafo direcionado simples de citações entre documentos.

| Método | Descrição |
|--------|-----------|
| `add_edge(citing_id, cited_id)` | Registra citação |
| `add_document_edges(doc_id, references)` | Registra citações a partir de lista de referências |
| `get_related(doc_id, max_depth=1)` | Retorna docs relacionados (citados + citantes) |
| `expand_retrieval(evidence_list, max_depth=1)` → `List[str]` | Expands doc_ids via grafo |
| `get_citation_network(doc_id)` → `Dict` | Rede de citações de um documento |

### C3 — OutlineSynthesizer
Síntese de resposta guiada por outline.

| Método | Descrição |
|--------|-----------|
| `plan(query)` → `List[OutlineSection]` | Gera outline a partir da consulta |
| `retrieve_per_section(outline, rag)` → `Dict[str, List]` | Evidências por seção |
| `synthesize(outline, section_evidence)` → `Dict` | Resposta estruturada |

`OutlineSection`:
- `section_id`: str
- `title`: str
- `focus_query`: str — sub-consulta para retrieval
- `expected_content`: str — o que a seção deve cobrir

### C4 — RAGEvolved (orquestrador)

| Método | Descrição |
|--------|-----------|
| `analyze_complexity(query)` → `str` | Delega para AdaptiveRetriever |
| `retrieve(query, top_k=10)` → `List[RetrievedEvidence]` | Retrieval adaptativo + expansão de citações |
| `answer(query)` → `Dict` | Resposta completa: retrieve → expand → synthesize |
| `answer_simple(query)` → `Dict` | Resposta direta (sem outline) |
| `answer_structured(query)` → `Dict` | Resposta com outline e seções |

## Critérios de Aceitação (SDD Gate)

### CA1 — Módulo e estrutura
- [ ] `rag/evolved.py` define `RAGEvolved`, `AdaptiveRetriever`, `CitationGraph`, `OutlineSynthesizer`
- [ ] Dependências: apenas `json`, `logging`, `dataclasses`, `statistics`, `typing`, `re`, `time`
- [ ] `RAGEvolved` aceita `ScientificRAG` como base (composição, não herança)

### CA2 — AdaptiveRetriever
- [ ] `analyze_complexity("What is X?")` retorna "simple"
- [ ] `analyze_complexity("Compare X and Y across Z dimensions")` retorna "moderate" ou "complex"
- [ ] `analyze_complexity("Synthesize findings from A, B, C regarding D and discuss implications for E")` retorna "complex"
- [ ] `retrieve_adaptive()` para queries "simple" usa estratégia sequential (1 chamada ao RAG)
- [ ] `retrieve_adaptive()` para queries "complex" usa estratégia parallel (2+ sub-queries)
- [ ] Resultados de parallel retrieval fundem evidências, deduplicam por chunk_id, ordenam por score

### CA3 — CitationGraph
- [ ] `add_edge()` registra aresta dirigida entre citing e cited
- [ ] `add_document_edges()` processa lista de referências
- [ ] `get_related()` retorna citados + citantes até max_depth
- [ ] `expand_retrieval()` retorna doc_ids expandidos além dos evidence originais
- [ ] Grafo vazio não causa erro nas operações

### CA4 — OutlineSynthesizer
- [ ] `plan("What is the impact of X on Y?")` retorna outline com 2+ seções
- [ ] Cada OutlineSection tem section_id, title, focus_query, expected_content
- [ ] `retrieve_per_section()` retorna dict com section_id → List[evidence]
- [ ] `synthesize()` retorna dict com sections, each contendo title, evidence_summary, key_findings
- [ ] Sections sem evidência têm summary "No evidence retrieved for this section"

### CA5 — RAGEvolved
- [ ] `analyze_complexity()` delega para AdaptiveRetriever
- [ ] `retrieve()` retorna lista de RetrievedEvidence com deduplicação
- [ ] `answer_simple()` para queries simples retorna resposta direta (abstém se sem evidência)
- [ ] `answer_structured()` para queries complexas retorna estrutura com outline + seções
- [ ] `answer()` roteia automaticamente entre simple/structured baseado na complexidade

### CA6 — Integração com RAG existente
- [ ] `RAGEvolved` usa `ScientificRAG` como motor de indexação/retrieval base
- [ ] `ScientificRAG` permanece inalterado (SPEC-919 intacto)
- [ ] Testes SPEC-919 continuam passando

### CA7 — Fallback
- [ ] Sem CitationGraph populado, expand_retrieval retorna lista vazia
- [ ] Para queries sem evidência, answer() abstém com mensagem clara
- [ ] Todos os componentes operam offline sem dependências externas

- [ ] Testes do AdaptiveRetriever (complexity analysis + strategies)
