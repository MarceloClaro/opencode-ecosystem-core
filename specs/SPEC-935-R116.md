# SPEC-935-R116 — Instalação Multiplataforma, Ícone Próprio, Claude CLI e Manual/Helpdesk

## Objetivo

Melhorar a instalação do OpenCode CLI, Antigravity CLI e (novo) Claude Code CLI em todas as plataformas (Windows/Linux/macOS), personalizar os ícones dos atalhos com uma identidade visual própria, e tornar o `marceloclaro` CLI mais intuitivo e prático com um manual e um helpdesk (diagnóstico guiado).

## Motivação

Uma investigação do instalador existente (`installer/windows/`) e da camada de integração de CLIs encontrou bugs reais que impediam o funcionamento correto do que já estava documentado:

- `AntigravityBridge` procurava o binário `antigravity`, mas o binário real instalado se chama **`agy`** — a ponte nunca detectava disponibilidade.
- `provision.sh` já chamava `integrations.opencode_cli.OpenCodeCLIIntegration('.').generate_config()`, uma classe que **não existia** — o passo de regeneração do `opencode.json` falhava silenciosamente.
- `opencode.json.instructions` apontava para `AGENTS.md`, que não existia.
- O README linkava um "Guia Manual" para Linux/macOS que apontava para `ARCHITECTURE.md`, sem nenhuma seção de instalação.
- Não existia nenhum ícone/logo próprio do projeto — os atalhos usavam ícones genéricos do sistema (`wsl.exe`, `cmd.exe`).
- Não existia nenhuma integração com Claude Code CLI, nem instalador nativo para Linux/macOS.
- Não existia manual de usuário nem helpdesk — toda a documentação era voltada a desenvolvedores, e o `marceloclaro/cli.py` expunha jargão interno (Blackboard, MetaBus) sem tradução nem opção de ajuda.
- `doctor()` não verificava se as CLIs externas estavam instaladas.
- `build_livro_tritemo.py::copy_to_desktop` tinha um path hardcoded a um usuário específico do Windows.

## Escopo

### Em escopo
- Correção dos 5 bugs reais acima (`integrations/antigravity/bridge.py`, `integrations/opencode_cli.py`, `AGENTS.md`, `build_livro_tritemo.py`, `marceloclaro/catalog_loader.py`)
- Ícone próprio gerado por script (`assets/generate_icon.py`), usado nos 4 atalhos de desktop do Windows
- Claude Code CLI como opção de primeira classe: instalação (`installer/common/install_clis.sh`), atalho de desktop, check em `doctor()`, `CLAUDE.md` reescrito como manual
- Instalador nativo Linux (`installer/linux/install.sh`) e macOS best-effort (`installer/macos/install.sh`)
- Desinstaladores para as 3 plataformas, com confirmação explícita para ações destrutivas (remover WSL, remover CLIs compartilhadas, apagar o repositório)
- `MANUAL.md` (linguagem simples) e `marceloclaro/helpdesk.py` (doctor + sugestões), integrados ao `marceloclaro/cli.py` (itens `[7]`/`[8]`)

### Fora de escopo (documentado, não escondido)
- Geração real de `.icns` (exige macOS físico) — só o script auxiliar (`installer/macos/build_icns.sh`) fica pronto
- `.app` bundle completo para macOS — usa-se `.command` como alternativa prática
- Testar de fato em macOS/distros não-Debian — best-effort, marcado como tal

## Critérios de Aceitação

1. `OpenCodeCLIIntegration('.').generate_config()` funciona (corrige o bug do `provision.sh`)
2. `AntigravityBridge()` aponta para `agy` por padrão
3. Nenhuma descrição de agente do catálogo é truncada em `"<!--"`
4. `assets/generate_icon.py` produz `icon.png`/`icon.ico` válidos, usados nos atalhos do instalador Windows
5. `installer/common/install_clis.sh` instala as 4 CLIs (OpenCode, Antigravity, Claude Code, Ollama) e é reutilizado por Windows (WSL), Linux e macOS
6. `doctor()` inclui um check `external_clis` (sempre `warn`, nunca `fail`, quando ausente)
7. `helpdesk()` mapeia cada check `warn`/`fail` do doctor para uma sugestão em linguagem simples
8. `marceloclaro/cli.py` tem itens `[7] Ajuda`/`[8] Helpdesk` e comandos diretos `ajuda`/`helpdesk`
9. Instaladores e desinstaladores das 3 plataformas têm sintaxe bash válida (`bash -n`)
10. Desinstaladores exigem confirmação explícita (`CONFIRMO`) antes de qualquer ação destrutiva (remover distro WSL, desativar WSL, remover CLIs compartilhadas, apagar repositório)
11. `MANUAL.md`, `AGENTS.md`, `installer/README.md` existem com conteúdo mínimo; `README.md` não referencia mais o link quebrado

## Estratégia de Verificação

- `tests/test_r116_installer_platform_upgrade.py` (32 testes)
- `bash -n` em todos os scripts de instalação/desinstalação
- Execução manual: `python3 -m marceloclaro.cli doctor`, `python3 -m marceloclaro.cli helpdesk`, `python3 assets/generate_icon.py`
- `python3 -m pytest tests/ -q` completo (zero regressões)

## Riscos

- Os instaladores Linux/macOS não foram executados de ponta a ponta em ambiente limpo real (só validados sintaticamente e por inspeção) — best-effort declarado explicitamente para macOS
- A detecção de usuário Windows a partir do WSL (`_detect_windows_desktop_from_wsl`) depende de `cmd.exe` estar acessível e responder em uma das codificações testadas (utf-8/cp850/cp1252); ambientes muito atípicos podem cair no fallback de criar `~/Desktop` dentro do próprio WSL

## Entrega Esperada

Instalação consistente das 3 CLIs (OpenCode, Antigravity, Claude Code) mais Ollama nas 3 plataformas, com identidade visual própria, desinstalação segura por padrão (destrutivo só com confirmação explícita), e uma camada de manual/helpdesk que traduz o jargão interno do ecossistema para quem só quer usá-lo.
