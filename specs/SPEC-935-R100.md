# SPEC-935-R100: MCP Security Hardening

## Objetivo
Adicionar camadas de segurança ao MCP Server (R94): **validação de entrada (Guard Model)**, **auditoria de chamadas (Audit Trail)**, **detecção de tool poisoning** e **rate limiting** — alinhado à taxonomia de ameaças MCP (Hou et al., 2025) e protocolo MCIP.

## Componentes

### C1 — MCPGuard
Intercepta chamadas de ferramentas, valida argumentos contra schemas, verifica permissões.

### C2 — AuditLogger
Registro estruturado de todas as operações MCP com timestamp, caller, ferramenta, parâmetros e resultado.

### C3 — ToolVetter
Detecta padrões suspeitos nos argumentos: injeção de prompt, command injection, path traversal.

### C4 — RateLimiter
Token bucket simples por caller identity (IP ou caller_id).

## Critérios de Aceitação

- CA1: MCPGuard valida argumentos contra JSON Schema e rejeita tipos inválidos
- CA2: AuditLogger registra timestamp, tool, args (sanitizados), resultado, duração
- CA3: ToolVetter detecta padrões de prompt injection (ex: "ignore instructions")
- CA4: ToolVetter detecta command injection (ex: "; rm -rf /")
- CA5: ToolVetter detecta path traversal (ex: "../../../etc/passwd")
- CA6: RateLimiter permite até N chamadas por janela, bloqueia excesso
- CA7: Integração: MCPGuard envolvendo handlers do MCP Server
- CA8: Falha de segurança não quebra o servidor (graceful rejection)
- CA9: 15+ testes TDD
