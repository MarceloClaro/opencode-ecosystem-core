# SPEC-935-R98: Academic Novelty V2 — OpenNovelty-Style

## Objetivo
Substituir a heurística simples de novidade do R93 por um pipeline granular de extração de contribution points, retrieval multi-source, comparação ponto-a-ponto e taxonomia hierárquica — alinhado ao estado da arte (OpenNovelty, Zhang et al., 2026).

## Motivação
O `NoveltyAnalyzer` atual (R93) tem 4 limitações críticas:
1. **Score por keyword heurística** — originalidade medida por palavras-chave no título
2. **Busca genérica** — uma única query para toda a tese, ignorando contribuições específicas
3. **Mapa de contribuições template** — sempre "Novel integration of X", sem granularidade
4. **Sem evidência explícita** — posicionamento genérico sem citações específicas

R98 resolve com pipeline OpenNovelty-style em 5 estágios.

## Arquitetura

```
NoveltyAnalyzerV2
│
├── 1. ContributionPointExtractor
│     Thesis → [Point(claim, type, evidence)]
│     Tipos: hypothesis_claim, methodological_claim,
│            finding_claim, framework_claim, application_claim
│
├── 2. PointwiseLiteratureRetriever
│     Para cada Point → busca literatura específica
│     [Point(claim)] → [Point(claim, related_works[])]
│
├── 3. PointwiseNoveltyScorer
│     Para cada Point → compara com related_works
│     → novelty_per_point: {point_id: {score, evidence, gaps}}
│
├── 4. HierarchicalTaxonomyBuilder
│     Related works → árvore hierárquica
│     → taxonomy_tree: {root → area → subarea → works}
│
└── 5. EvidencedNoveltyReport
      → Report com per-point scores, taxonomy, discrepancy analysis
```

## Componentes

### C1 — ContributionPointExtractor
Extrai pontos específicos de contribuição de uma tese.

| Método | Descrição |
|--------|-----------|
| `extract(thesis)` → `List[ContributionPoint]` | Extrai pontos de contribuição do texto da tese |
| `_extract_hypothesis_claim(thesis)` → `ContributionPoint` | Claim da hipótese central |
| `_extract_methodological_claim(thesis)` → `ContributionPoint` | Claim metodológica |
| `_extract_framework_claim(thesis)` → `ContributionPoint` | Claim de framework/paradigma |
| `_extract_application_claim(thesis)` → `ContributionPoint` | Claim de aplicação/impacto |

`ContributionPoint`:
- `point_id`: str (ex: "cp_hypothesis_1")
- `type`: str (hypothesis_claim, methodological_claim, framework_claim, application_claim)
- `claim`: str — o texto da claim
- `keywords`: List[str] — palavras-chave da claim
- `confidence`: float (0-1) — confiança na extração

### C2 — PointwiseLiteratureRetriever
Busca literatura relacionada para cada contribution point individualmente.

| Método | Descrição |
|--------|-----------|
| `retrieve(points, thesis)` → `List[PointWithLiterature]` | Busca literatura para cada point |
| `_search_for_point(point, top_k=3)` → `List[RelatedWork]` | Busca específica por point |
| `_fallback_literature(point)` → `List[RelatedWork]` | Fallback quando busca falha |

`RelatedWork`:
- `title`: str
- `year`: int
- `relevance`: float (0-1)
- `contribution`: str — contribuição específica do trabalho
- `gap`: str — lacuna que o trabalho deixa
- `source`: str — origem (fallback / web)

### C3 — PointwiseNoveltyScorer
Calcula novidade ponto-a-ponto com evidência explícita.

| Método | Descrição |
|--------|-----------|
| `score(points_with_lit)` → `Dict[str, PointNovelty]` | Scores por ponto |
| `_score_single_point(point, related)` → `PointNovelty` | Score de um ponto específico |
| `_compute_global_score(point_scores)` → `float` | Score global ponderado |

`PointNovelty`:
- `point_id`: str
- `point_type`: str
- `novelty_score`: float (0-100) — quão original é este ponto
- `evidence`: str — explicação de por que é (ou não) original
- `overlapping_works`: List[str] — títulos de trabalhos que já abordam o ponto
- `gap_identified`: str — lacuna específica que este ponto preenche

### C4 — HierarchicalTaxonomyBuilder
Constrói taxonomia hierárquica de trabalhos relacionados.

| Método | Descrição |
|--------|-----------|
| `build(points_with_lit)` → `TaxonomyTree` | Constrói árvore a partir dos pontos |
| `_group_by_area(works)` → `Dict[str, List]` | Agrupa por área temática |
| `_build_tree(groups)` → `dict` | Constrói estrutura hierárquica |

`TaxonomyTree`:
- `root`: str — tema central
- `children`: `List[TaxonomyNode]` — áreas

`TaxonomyNode`:
- `name`: str — nome da área
- `relevance`: float (0-1)
- `works`: List[RelatedWork] — trabalhos nesta área
- `children`: List[TaxonomyNode] — sub-áreas

### C5 — EvidencedNoveltyReport
Relatório final enriquecido.

`EvidencedNoveltyReport`:
- `thesis_id`: str
- `thesis_title`: str
- `global_novelty_score`: float (0-100)
- `per_point_scores`: Dict[str, PointNovelty]
- `taxonomy_tree`: TaxonomyTree
- `top_related_works`: List[RelatedWork]
- `discrepancy_analysis`: str — análise de discrepâncias entre claims e literatura
- `positioning_statement`: str — statement de posicionamento
- `methodology_novelty`: float — sub-score metodológico
- `framework_novelty`: float — sub-score de framework
- `application_novelty`: float — sub-score de aplicação
- `analyzed_at`: str

## Critérios de Aceitação (SDD Gate)

### CA1 — Módulo e estrutura
- [ ] `synthetic_university/novelty_v2.py` define `NoveltyAnalyzerV2` e dataclasses associadas
- [ ] Dependências: apenas `json`, `logging`, `os`, `dataclasses`, `statistics`, `typing`, `re`, `time`
- [ ] `NoveltyAnalyzerV2` aceita `lang` e `memory` (Optional[EvolutionaryMemorySubstrate])

### CA2 — ContributionPointExtractor
- [ ] `extract()` retorna lista de ContributionPoint com types hypothesis_claim, methodological_claim, framework_claim, application_claim
- [ ] Para thesis sem abstract, extract() retorna 1 ponto por conceito (fallback)
- [ ] Cada ContributionPoint tem point_id único, claim text, keywords, confidence
- [ ] Mínimo 2 pontos retornados para thesis com hypothesis + methodology

### CA3 — PointwiseLiteratureRetriever
- [ ] `retrieve()` retorna lista de PointWithLiterature com points + related_works
- [ ] Cada related_work tem title, year, relevance, contribution, gap, source
- [ ] Sempre retorna fallback literature quando busca real falha
- [ ] Fallback gera 2-3 works por point com base nas keywords do point

### CA4 — PointwiseNoveltyScorer
- [ ] `score()` retorna dict com point_id → PointNovelty
- [ ] Cada PointNovelty tem novelty_score (0-100), evidence, overlapping_works, gap_identified
- [ ] Score global computado como média ponderada dos per-point scores
- [ ] Pontos do tipo methodological_claim têm peso 0.30 no global

### CA5 — HierarchicalTaxonomyBuilder
- [ ] `build()` retorna TaxonomyTree com root theme + children areas
- [ ] Cada TaxonomyNode tem name, relevance, works, children
- [ ] Árvore tem pelo menos 2 áreas quando há 3+ points
- [ ] works vazios não quebram a construção

### CA6 — EvidencedNoveltyReport
- [ ] `analyze(thesis)` retorna EvidencedNoveltyReport completo
- [ ] Report inclui global_novelty_score, per_point_scores, taxonomy_tree, top_related_works
- [ ] Report inclui discrepancy_analysis textual
- [ ] Report inclui positioning_statement citando trabalhos específicos

### CA7 — Integração com R97 (Evolutionary Memory)
- [ ] Quando `memory` é fornecido, scores de novidade são registrados no StagnationDetector
- [ ] Quando `memory` é fornecido, contribution points são registrados no IdeationMemory
- [ ] Falha do memory não interrompe a análise

### CA8 — Fallback sem dependências externas
- [ ] `analyze()` funciona sem OpenCode CLI, sem rede, sem LLM
- [ ] Fallback gera dados sintéticos mas estruturalmente válidos
- [ ] Todos os campos do report são preenchidos mesmo em fallback
