# SPEC-935-R120 — Comando `pesquisa` no CLI + visibilidade do fallback Sci-Hub

## Objetivo

Expor o pipeline de pesquisa científica multiplataforma
(`MarceloClaroOrchestrator.research()`, SPEC-017/R111 — 11 fontes
acadêmicas incluindo PubMed/bioRxiv/CORE) diretamente no CLI interativo
do `marceloclaro` (`marceloclaro/cli.py`), que até aqui só permitia
acioná-lo programaticamente via Python. Em paralelo, tornar visível na
arquitetura documentada (README/ARCHITECTURE/MANUAL) e no `doctor()` o
uso já existente de `scihub-cli` como fallback de download de PDF para
artigos pagos.

## Motivação

Usuário perguntou se o Research Hub (11 fontes, R111) estava integrado
à arquitetura. Confirmado que sim, de ponta a ponta
(`research/searchers.py` → `MultiSearcher` → `research/hub.py::
ResearchHub` → `marceloclaro/orchestrator.py::research()`/
`research_search()`), mas com uma lacuna real: `marceloclaro/cli.py`
não expunha nenhum comando para isso — só acessível programaticamente.

Em seguida, o usuário pediu para adicionar Sci-Hub. Investigação
mostrou que o fallback via `scihub-cli` (pacote pip de terceiros,
Oxidane-bot) **já existia** em `research/downloader.py` desde antes
desta sessão, para download de artigos pagos sem acesso aberto direto —
só não estava documentado nos arquivos de arquitetura nem checado pelo
`doctor()`, ao contrário das demais CLIs externas opcionais
(OpenCode/Antigravity/Claude Code/Ollama, R116).

## Escopo

### Em escopo
- `marceloclaro/cli.py`:
  - Comando direto `pesquisa "<tema>"` (alias `research`), com flags
    `--max-papers N`, `--platforms a,b,c`, `--no-download`
  - Opção `[9]` no menu interativo, com prompts guiados (tema, número
    máximo de artigos, baixar PDFs?) e resumo amigável ao final
  - `_parse_pesquisa_flags()`: parsing isolado e testável das flags
  - `AJUDA_TEXT`/`MENU` atualizados, incluindo menção ao fallback
    Sci-Hub
- `marceloclaro/doctor.py::EXTERNAL_CLIS`: adicionado `scihub-cli` →
  `pip install scihub-cli` (reaproveita o `_check_external_clis()` já
  existente, sempre `warn` nunca `fail`, mesmo padrão de R116)
- Documentação: `README.md` e `ARCHITECTURE.md` (os dois diagramas
  Mermaid, mantidos sincronizados desde o achado do R117) — nó Research
  Hub ganha menção ao CLI `pesquisa` (R120) e ao fallback Sci-Hub;
  `MANUAL.md` ganha a opção `[9]` na tabela do menu e nota na FAQ de
  CLIs opcionais

### Fora de escopo
- Qualquer scraping direto de domínios Sci-Hub — o acesso continua
  inteiramente delegado ao pacote de terceiros já existente
  (`scihub-cli`), como já era antes desta sessão
- Interface web/dashboard para a pesquisa (só CLI, como as demais
  opções do menu)

## Critérios de Aceitação

1. `python3 -m marceloclaro.cli pesquisa "<tema>"` (e alias `research`)
   chama `orchestrator.research()` e imprime o manifest em JSON
2. Flags `--max-papers`, `--platforms`, `--no-download` são repassadas
   corretamente
3. Sem tema, imprime uso e sai com código 1 (sem chamar `research()`)
4. Opção `[9]` do menu interativo funciona com prompts guiados e nunca
   chama `research()` se o tema for vazio
5. `scihub-cli` aparece em `EXTERNAL_CLIS` e é reportado como `warn`
   (nunca `fail`) quando ausente
6. Nenhum teste faz chamada de rede real — `orchestrator.research()`
   sempre mockado
7. Suíte de testes completa permanece verde (exceto a falha
   intermitente pré-existente `test_failed_task_lowers_confidence`)

## Estratégia de Verificação

- `tests/test_r120_cli_pesquisa_command.py` (13 testes): parsing de
  flags, comando direto (com e sem tema, alias, flags), menu
  interativo (opção 9 com e sem tema), registro e comportamento do
  check `scihub-cli` no doctor
- Execução manual: `python3 -m marceloclaro.cli ajuda` (confere o novo
  texto) e `run_doctor()` (confere o novo check)
- `python3 -m pytest tests/ -q` completo
