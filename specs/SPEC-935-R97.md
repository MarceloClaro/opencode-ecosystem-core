# SPEC-935-R97: Evolutionary Memory Substrate

## Objetivo
Adicionar um substrato de memória persistente evolutiva ao ecossistema, permitindo que o `ContinuousDiscoveryLoop` e o `NoveltyAnalyzer` aprendam com ciclos passados — evitando repetição de falhas, reutilizando estratégias bem-sucedidas, detectando estagnação e refletindo periodicamente sobre o progresso.

## Motivação
Atualmente o `ContinuousDiscoveryLoop` (R95) gera teses a cada ciclo sem memória do que já foi tentado, quais direções falharam e quais estratégias funcionaram. Isso leva a:

1. **Repetição de erros** — a mesma direção de pesquisa infrutífera é revisitada em ciclos subsequentes.
2. **Ausência de aprendizado** — estratégias de ideação que funcionaram bem são esquecidas.
3. **Estagnação invisível** — não há detecção de quando a novidade média está estagnada há vários ciclos.
4. **Falta de reflexão** — não há consolidação periódica de lições aprendidas.

R97 resolve isso com 4 componentes integrados, inspirados em EvoScientist (Lyu et al., 2026) e CORAL (Mishra et al., 2026).

## Arquitetura

```
┌─────────────────────────────────────────────────────────┐
│              EvolutionaryMemorySubstrate                │
│  ┌─────────────────┐  ┌──────────────────────────────┐  │
│  │  IdeationMemory  │  │  ExperimentationMemory       │  │
│  │  • direções      │  │  • estratégias de código     │  │
│  │  • outcomes      │  │  • efetividade histórica     │  │
│  │  • fail/success  │  │  • reuso por contexto        │  │
│  └──────┬───────────┘  └──────────┬───────────────────┘  │
│         │                         │                      │
│  ┌──────┴─────────────────────────┴──────────────┐       │
│  │           HeartbeatReflection                  │       │
│  │  • a cada N ciclos, consolida lições           │       │
│  │  • atualiza scores de confiança                │       │
│  │  • gera relatório de reflexão                  │       │
│  └──────────────────────┬─────────────────────────┘       │
│                         │                                  │
│  ┌──────────────────────┴─────────────────────────┐       │
│  │           StagnationDetector                    │       │
│  │  • monitora novidade média ao longo do tempo    │       │
│  │  • detecta platô (M ciclos sem melhoria)        │       │
│  │  • sugere redirecionamento de foco              │       │
│  └─────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────┘
         │                        │
         ▼                        ▼
  ContinuousDiscoveryLoop    NoveltyAnalyzer
         (R95)                   (R93)
```

## Componentes

### C1 — IdeationMemory
Memória de direções de pesquisa já exploradas, com outcomes.

| Método | Descrição |
|--------|-----------|
| `record_idea(direction, outcome, score, metadata)` | Registra direção explorada com resultado |
| `get_promising_directions(top_k=5, min_score=60)` | Retorna direções com score ≥ threshold |
| `get_failed_directions(max_score=40)` | Retorna direções que não funcionaram |
| `is_already_explored(idea, threshold=0.7)` | Verifica similaridade com ideias existentes |
| `get_direction_stats(direction_id)` | Estatísticas de uma direção (n_tentativas, avg_score, trend) |
| `get_exploration_summary()` | Sumário de exploração (total, success_rate, diversity) |

### C2 — ExperimentationMemory
Memória de estratégias de ideação/código que funcionaram.

| Método | Descrição |
|--------|-----------|
| `record_strategy(strategy_id, context, effectiveness, metadata)` | Registra estratégia usada em dado contexto |
| `get_reusable_strategies(context, top_k=3)` | Recupera estratégias relevantes para o contexto |
| `get_effectiveness_stats(strategy_id)` | Efetividade histórica (mean_score, n_uses, trend) |
| `get_strategy_portfolio()` | Portfólio completo com rankings |

### C3 — HeartbeatReflection
Reflexão periódica que consolida aprendizado.

| Método | Descrição |
|--------|-----------|
| `should_reflect(cycle_count, interval=5)` | Verifica se é hora de refletir |
| `reflect(memory_substrate)` | Executa ciclo de reflexão |
| `get_reflection_history()` | Histórico de reflexões realizadas |
| `get_insights()` | Síntese das lições aprendidas |

### C4 — StagnationDetector
Detector de platô de inovação.

| Método | Descrição |
|--------|-----------|
| `record_novelty_score(cycle, score)` | Alimenta série temporal de novidade |
| `is_stagnated(window=5, threshold=0.05)` | Detecta platô nos últimos N ciclos |
| `get_stagnation_report()` | Relatório detalhado de estagnação |
| `suggest_redirection()` | Sugere direções alternativas com base em falhas passadas |
| `get_novelty_trend()` | Tendência de novidade (improving/stable/declining) |

## Critérios de Aceitação (SDD Gate)

### CA1 — Módulo e estrutura
- [ ] `synthetic_university/evolutionary_memory.py` define a classe `EvolutionaryMemorySubstrate`
- [ ] `EvolutionaryMemorySubstrate` compõe os 4 componentes (IdeationMemory, ExperimentationMemory, HeartbeatReflection, StagnationDetector)
- [ ] Nenhuma dependência externa além de `json`, `logging`, `os`, `dataclasses`, `statistics`, `typing`

### CA2 — IdeationMemory
- [ ] `record_idea()` armazena direção com outcome (success/failure), score (0-100), timestamp e metadados
- [ ] `get_promising_directions()` retorna apenas direções com score ≥ min_score, ordenadas por score descendente
- [ ] `get_failed_directions()` retorna direções com score ≤ max_score
- [ ] `is_already_explored()` retorna True se similaridade coseno (Título) ≥ threshold com alguma ideia existente
- [ ] `get_direction_stats()` retorna dict com `n_attempts`, `avg_score`, `best_score`, `trend` (improving/stable/declining)
- [ ] `get_exploration_summary()` retorna dict com `total_ideas`, `success_rate`, `avg_score`, `diversity` (conceitos únicos)

### CA3 — ExperimentationMemory
- [ ] `record_strategy()` armazena estratégia com context tags, effectiveness (0-100) e timestamp
- [ ] `get_reusable_strategies()` retorna estratégias ordenadas por effectiveness, filtradas por overlap de contexto
- [ ] `get_effectiveness_stats()` retorna dict com `mean_effectiveness`, `n_uses`, `max_effectiveness`, `trend`
- [ ] `get_strategy_portfolio()` retorna lista completa rankeada por `mean_effectiveness`

### CA4 — HeartbeatReflection
- [ ] `should_reflect(cycle_count, interval=5)` retorna True quando cycle_count % interval == 0
- [ ] `reflect()` gera reflection_id único, consolida insights de ideation + experimentation, registra timestamp
- [ ] `get_reflection_history()` retorna lista de reflexões ordenadas por timestamp
- [ ] `get_insights()` retorna síntese textual dos insights mais recentes

### CA5 — StagnationDetector
- [ ] `record_novelty_score()` aceita cycle_index (int) e score (float) e armazena em série temporal
- [ ] `is_stagnated(window=5, threshold=0.05)` retorna True se variância dos últimos `window` scores < `threshold`
- [ ] `get_stagnation_report()` retorna dict com `is_stagnated`, `window`, `variance`, `mean`, `recommendation`
- [ ] `suggest_redirection()` retorna lista de direções de falha passada como sugestão de exploração alternativa
- [ ] `get_novelty_trend()` retorna "improving", "stable" ou "declining" baseado na inclinação da regressão linear

### CA6 — Integração com ContinuousDiscoveryLoop (R95)
- [ ] `ContinuousDiscoveryLoop` aceita parâmetro `memory` (Optional[EvolutionaryMemorySubstrate])
- [ ] Quando `memory` é fornecido, `run_cycle()` consulta `IdeationMemory.is_already_explored()` antes de gerar teses
- [ ] Quando `memory` é fornecido, `run_cycle()` registra outcomes no `IdeationMemory` e `ExperimentationMemory` após cada ciclo
- [ ] Quando `memory` é fornecido, `run_cycle()` chama `HeartbeatReflection.should_reflect()` e `reflect()` no final
- [ ] Quando `memory` é fornecido, `run_cycle()` verifica `StagnationDetector.is_stagnated()` e inclui alerta no resultado

### CA7 — Testes TDD
- [ ] Mínimo 15 testes em `tests/test_r97_evolutionary_memory.py`
- [ ] Testes do `IdeationMemory`: record, retrieve, avoid duplicates, direction stats, summary
- [ ] Testes do `ExperimentationMemory`: record, retrieve by context, effectiveness stats, portfolio
- [ ] Testes do `HeartbeatReflection`: should_reflect, reflect execution, history, insights
- [ ] Testes do `StagnationDetector`: record, detect plateau, suggest redirection, trend
- [ ] Testes de integração com `ContinuousDiscoveryLoop`
- [ ] Testes simulados (sem rede externa, sem IO real desnecessário)
- [ ] Todos os testes passam com `pytest -q tests/test_r97_evolutionary_memory.py`

### CA8 — Serialização e persistência
- [ ] `EvolutionaryMemorySubstrate.save(path)` exporta estado completo para JSON
- [ ] `EvolutionaryMemorySubstrate.load(path)` class method carrega estado de arquivo JSON
- [ ] Formato JSON inclui versão do schema, timestamp, e todos os componentes

### CA9 — Integração com MetaBus
- [ ] Quando `reflection_mode=True`, `HeartbeatReflection.reflect()` publica evento no MetaBus se disponível
- [ ] Falha na publicação não interrompe a reflexão

## Dependências
- `synthetic_university.continuous_discovery` (R95) — integração
- `mci.metabus` (opcional — MetaBus publish)

## Não Escopo
- R97 **não** implementa schedule/threading real para heartbeat
- R97 **não** modifica o MCP Server (R94) ou API Gateway (R96)
- R97 **não** implementa busca web ou LLM calls

## Critério de Sucesso
Após R97, o ecossistema DEVE:
- Evitar gerar teses com direções já falhadas (CA6.2)
- Detectar estagnação de novidade em até 5 ciclos (CA5.2)
- Manter histórico persistente entre execuções (CA8)
- Refletir a cada 5 ciclos com insights consolidados (CA4)
