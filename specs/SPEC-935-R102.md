# SPEC-935-R102 — Deep Research Agent

## Objetivo

Implementar um agente de pesquisa profunda (Deep Research) com orquestração hierárquica,
combinação de estratégias Breadth-First (BF) e Depth-First (DF), e construção incremental
de um Evidence Graph para síntese multi-fontes de evidência científica.

**Base teórica (6 papers 2025–2026):**

1. **DeepEvidence** (Wang et al., *Nature Machine Intelligence*, July 2026) — Sistema
   multiagente hierárquico com Orchestrator, BFRS (exploração larga), DFRS (investigação
   profunda) e Evidence Graph incremental sobre 15+ bases biomédicas.
   ([Nature MI](https://www.nature.com/articles/s42256-026-01266-0),
   [arXiv:2601.11560](https://arxiv.org/abs/2601.11560))

2. **Super Research** (arXiv 2603.00582, 2026) — Decomposição estruturada + Super Wide
   Retrieval + Super Deep Investigation com auditagem baseada em grafo.

3. **DualGraph** (Shi et al., arXiv 2602.13830, 2026) — Arquitetura de dois grafos:
   Outline Graph (como escrever) + Knowledge Graph (o que sabe), separando exploração
   de estrutura de relatório.

4. **A2RAG** (arXiv 2601.21162, 2026) — Controle adaptativo com verificação de suficiência
   de evidência e recuperador agentivo com escalonamento progressivo (local → bridge → global).

5. **DuMate-DeepResearch** (Yan et al., arXiv 2606.07299, 2026) — Planejamento dinâmico
   baseado em grafo, execução recursiva dois-níveis, otimização test-time com rubrica.

6. **Agents-K1** (Cao et al., arXiv 2606.13669, 2026) — Orquestração de conhecimento
   científico com parser multimodal 5-módulos, extração claims/evidência/métodos.

## Componentes

### 1. EvidenceGraph (core)

Grafo incremental que acumula conhecimento entre rodadas de pesquisa:

| Elemento | Descrição |
|----------|-----------|
| `Entity` | Nó (gene, doença, droga, paper, conceito) com tipo e metadados |
| `Relation` | Aresta tipada (TREATS, CAUSES, ASSOCIATED_WITH, CITES) |
| `Evidence` | Fragmento textual de suporte com fonte e score de confiança |
| `Provenance` | URL/DOI + timestamp + agente que registrou |

Operações: `add_entity`, `add_relation`, `expand_entity`, `find_paths`, `subgraph_query`, `export_visualization`

### 2. BFRS Agent (Breadth-First ReSearch)

- Explora vizinhança imediata de entidades semente no grafo
- Query paralela a múltiplas fontes (PubMed, OpenTargets, ClinicalTrials, etc.)
- Retorna conjunto amplo de candidatos para o Orchestrator
- Limitado por `max_breadth` e `max_sources`

### 3. DFRS Agent (Depth-First ReSearch)

- Segue cadeias multi-hop profundas (ex: gene → pathway → disease → drug → trial)
- Traça referências e citações recursivamente até `max_depth`
- Extrai evidência detalhada com contexto
- Usa o Evidence Graph para evitar ciclos e re-exploração

### 4. OrchestratorAgent

- **Planejamento**: Decompõe pergunta em research plan com sub-perguntas
- **Roteamento**: Decide BF vs DF vs híbrido baseado na sub-pergunta
- **Suficiência**: Avalia se evidência acumulada responde à pergunta (gate)
- **Síntese**: Gera relatório final com citações e grafo de evidência
- **Controle**: Gerencia orçamento (rodadas, ações, tempo)

### 5. ExecutionSandbox

- Ambiente Python isolado para execução programática
- Query APIs científicas, análise downstream, verificação cruzada
- Cache de resultados para eficiência
