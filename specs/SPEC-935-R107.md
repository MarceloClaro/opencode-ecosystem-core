# SPEC-935-R107 — Auditoria Sistêmica e Hardening de Contratos

## Objetivo

Executar uma auditoria técnica do ecossistema **OpenCode Ecosystem Core** com foco em erros de runtime, incompatibilidades entre wrappers e APIs reais, e fragilidades de utilitários de qualidade.

O ciclo R107 deve corrigir os pontos de maior impacto operacional sem regressões, priorizando:

1. Alinhamento dos wrappers da webapp (`webapp/pipeline_helpers.py`) com as APIs reais de `agentic_science_v2`
2. Correção do browser/resumo do Evidence Graph em `webapp/consultation_helpers.py`
3. Robustez do parser de saída do pytest em `scripts/quality_report.py` e `scripts/check_coverage.py`
4. Cobertura de testes para os contratos corrigidos

## Motivação

O ecossistema possui boa cobertura de testes globais, porém a auditoria revelou fragilidades relevantes fora da superfície mais coberta:

- Wrappers Streamlit apontando para métodos/parâmetros inexistentes
- Uso incorreto da API pública de `EvidenceGraph`
- Parsers de resultados do pytest que falham em cenários de sucesso total (`passed + skipped` sem `failed`)
- Lacunas de regressão em módulos recém-adicionados da webapp

## Escopo

### Em escopo

- `webapp/pipeline_helpers.py`
- `webapp/consultation_helpers.py`
- `scripts/quality_report.py`
- `scripts/check_coverage.py`
- novos testes de contrato / regressão

### Fora de escopo neste ciclo

- Reescrita completa do estado da webapp com `st.session_state`
- Refatoração total do `MarceloClaroOrchestrator`
- Revisão integral de todos os pacotes externos / legacy tests
- Mudanças arquiteturais amplas no pipeline acadêmico

## Critérios de Aceitação

1. `run_evosci()` usa a API real do R101 e retorna payload normalizado para a UI
2. `run_deep_research()` usa a API real do R102 e retorna payload normalizado para a UI
3. `run_peer_review()` usa a API real do R103 e adapta a saída para a UI sem chamar métodos inexistentes
4. `run_manuscript_revision()` adapta `review_package` e `manuscript` para o contrato real do R104d
5. `compose_paper()` deixa de usar parâmetros inexistentes e produz manuscrito formatado compatível com a UI
6. `query_evidence_graph()` e `get_evidence_graph_summary()` usam apenas a API pública de `EvidenceGraph`
7. `quality_report.py` e `check_coverage.py` interpretam corretamente saídas do pytest em cenários:
   - apenas `passed`
   - `passed + skipped`
   - `passed + failed + skipped`
8. Novos testes de regressão cobrem os contratos corrigidos
9. Suite principal permanece verde sem regressões

## Estratégia de Verificação

- Testes unitários para wrappers e parsers
- Smoke checks sintáticos/importáveis dos módulos alterados
- Execução de `pytest tests/ -q --tb=short`

## Riscos

- Divergência entre contratos esperados pela UI e payloads reais dos módulos científicos
- Dependência de payloads sintéticos nos wrappers exigindo adapters explícitos
- Possíveis efeitos colaterais em documentação ou exemplos que referenciem contratos antigos

## Entrega Esperada

Hardening do ecossistema com foco em confiabilidade operacional da webapp e dos scripts de qualidade, preservando compatibilidade com os fluxos atuais do Core.
