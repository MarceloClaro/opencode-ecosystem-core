# SPEC-935-R111 — Expansão de Fontes de Pesquisa (PubMed, bioRxiv, CORE)

## Objetivo

Expandir `research/searchers.py` de 8 para 11 fontes reais de busca
científica, adicionando PubMed (NCBI E-utilities), bioRxiv/medRxiv (via
filtro de prefixo DOI no Crossref) e CORE (API oficial, com chave).

## Motivação

Uma auditoria de capacidades alegadas vs. reais (ver `CORRIGENDUM.md`)
encontrou a alegação "busca em 10+ fontes acadêmicas (arXiv, PubMed,
OpenAlex)" — mas `research/searchers.py` tinha apenas 8 fontes, e PubMed
especificamente não estava entre elas. Em vez de corrigir a alegação para
baixo, implementamos as fontes que faltavam.

## Escopo

### Em escopo
- `PubMedSearcher`: NCBI E-utilities (`esearch` + `esummary`, 2 chamadas)
- `BiorxivSearcher`: Crossref filtrado por prefixo DOI `10.1101`
  (bioRxiv/medRxiv) — a API pública do bioRxiv não expõe busca por
  palavra-chave arbitrária, então reaproveitamos o mesmo padrão já usado
  por `ScieloSearcher`
- `CoreSearcher`: API oficial do CORE (`api.core.ac.uk/v3`), degrada
  graciosamente sem `CORE_API_KEY` configurada (mesmo padrão de
  `KaggleSearcher` para credenciais ausentes)

## Critérios de Aceitação

1. `ALL_SEARCHERS` contém 11 plataformas (as 8 anteriores + pubmed,
   biorxiv, core)
2. `PubMedSearcher` faz as 2 chamadas reais (esearch + esummary) e
   normaliza para `PaperRecord`
3. `BiorxivSearcher` usa `filter=prefix:10.1101` no Crossref
4. `CoreSearcher` retorna lista vazia (sem erro) quando `CORE_API_KEY`
   não está configurada
5. Falha de rede em qualquer um dos 3 novos buscadores degrada para
   lista vazia (via `BaseSearcher.search()`), sem propagar exceção
6. Testes offline (mockados) cobrem os 3 novos buscadores sem depender
   de rede real

## Estratégia de Verificação

- `tests/test_research.py` (testes novos: `TestPubMedSearcher`,
  `TestBiorxivSearcher`, `TestCoreSearcher`) — todos offline com
  `monkeypatch`
- Teste manual com rede real confirmando resultados reais do PubMed e
  bioRxiv, e degradação graciosa do CORE sem chave

## Riscos

- PubMed e bioRxiv (via Crossref) não requerem autenticação, mas estão
  sujeitos a rate limiting da NCBI/Crossref em uso intenso
- CORE só funciona com chave de API gratuita configurada pelo usuário
  (`CORE_API_KEY`) — sem isso, a fonte fica sempre vazia (documentado)
