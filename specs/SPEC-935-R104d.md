# SPEC-935-R104d — Agentic Manuscript Revision (Core Evolution R104)

## Objetivo

Implementar um sistema agentivo de revisão de manuscritos acadêmicos que:
1. Toma como entrada o **ReviewPackage** do R103 (peer review)
2. Mapeia cada crítica/claim para seções específicas do manuscrito
3. Gera propostas de revisão para cada ponto
4. Rastreia o estado de cada revisão (pendente → proposta → aplicada → verificada)
5. Produz um **RevisionReport** com resposta ponto-a-ponto (rebuttal letter)
6. Mantém integridade do manuscrito via diff controlado

**Base teórica (3 papers 2025–2026):**

1. **SWE-bench + Agentic Revision** — Adaptação do paradigma SWE-agent para revisão
   acadêmica: localizar → editar → verificar.

2. **ReViewGraph** (Li et al., arXiv 2511.08317, 2025) — Grafo de debates revisor-autor
   para rastrear revisões.

3. **DeepReviewer 2.0** (Weng et al., arXiv 2604.09590, 2026) — Revision package
   com verification agenda.

## Componentes

### 1. ReviewAnalyzer

Analisa o ReviewPackage do R103 e extrai:
- **Claims**: Afirmações do revisor sobre problemas/contribuições
- **Risks**: Nível de risco (Alto/Médio/Baixo)
- **Required Actions**: Ações de revisão necessárias
- **Critic Type**: Qual especialista gerou (Methodology/Results/Literature/Ethics)

### 2. SectionMapper

Mapeia claims para seções do manuscrito:
- Seção alvo (Introduction, Methods, Results, Discussion, etc.)
- Localização precisa (parágrafo, linha)
- Confiança do mapeamento (0.0–1.0)
- Sobreposições com outros claims

### 3. ProposalGenerator

Gera propostas de revisão:
- **original_text**: Texto original
- **revised_text**: Texto revisado proposto
- **rationale**: Justificativa da mudança
- **alternatives**: Lista de alternativas (mín. 2)
- **references**: Referências que suportam a mudança
- **risk_addressed**: Qual risco endereça

### 4. DiffEngine

Gerencia o diff controlado:
- Mantém `original` e `edited` versions
- Gera diff unificado
- Verifica integridade (não quebra estrutura do documento)
- Permite rollback por claim

### 5. OrchestratorRevision

Pipeline:
1. **Analyze**: Processa ReviewPackage → ReviewAnalyzer
2. **Map**: Mapeia claims → SectionMapper
3. **Propose**: Gera propostas → ProposalGenerator
4. **Apply**: Aplica revisões → DiffEngine
5. **Verify**: Verifica integridade e cobertura
6. **Report**: Gera RevisionReport + rebuttal letter
