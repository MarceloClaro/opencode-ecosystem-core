# SPEC-935-R95: Continuous Discovery Loop

## Objetivo
Criar um **watter/daemon** de descoberta contínua que periodicamente (1) enriquece teses do pipeline com busca web, (2) reavalia novidade, (3) gera novas correlações e (4) acumula resultados em um relatório evolutivo.

## Motivação
Atualmente cada módulo da Universidade Sintética (geração, enriquecimento, novidade) roda sob demanda. Não há um processo contínuo que mantenha as descobertas atualizadas à medida que nova literatura é incorporada. R95 preenche essa lacuna.

## Critérios de Aceitação (SDD Gate)

### CA1 — Estrutura do módulo
- [ ] Arquivo `synthetic_university/continuous_discovery.py` contém a classe `ContinuousDiscoveryLoop`
- [ ] Classe aceita `output_dir` (str), `interval_hours` (float, default 24), `lang` (str, default "en")
- [ ] Classe armazena um histórico de ciclos executados (`self.cycle_history: List[Dict]`)

### CA2 — Execução de ciclo único
- [ ] Método `run_cycle()` executa uma iteração completa:
  1. Gera 3-5 novas teses (usa `generate_theses()` interno com fallback)
  2. Enriquece cada tese (usa `ThesisEnricher.enrich()`)
  3. Reavalia novidade de cada tese enriquecida (usa `NoveltyAnalyzer.analyze()`)
  4. Calcula metadados do ciclo: timestamp, duração, número de teses, scores médios
- [ ] Retorna `Dict` com summary do ciclo

### CA3 — Acumulação histórica
- [ ] Cada ciclo é registrado em `self.cycle_history`
- [ ] Método `get_summary()` retorna:
  - `total_cycles`: int
  - `avg_novelty`: float
  - `avg_enrichment`: float (média de conceitos enriquecidos por tese)
  - `top_theses`: list das 5 melhores teses (por novelty score)
  - `latest_cycle`: timestamp do último ciclo

### CA4 — Exportação
- [ ] Método `export_report(cycle_index: int = -1)` salva relatório JSON em `{output_dir}/cycle_{n}.json`
- [ ] Método `export_all()` salva relatório consolidado em `{output_dir}/all_cycles.json`

### CA5 — Integração com run_cycle()
- [ ] `run_cycle()` deve funcionar sem LLM real (usa fallbacks dos módulos internos)
- [ ] Deve ser seguro chamar múltiplas vezes (não gerar efeitos colaterais em execuções paralelas)

### CA6 — Testes
- [ ] Mínimo 8 testes TDD em `tests/test_r95_continuous_discovery.py`
- [ ] Todos os testes devem passar sem rede externa nem LLM real
- [ ] Cobertura: criação, ciclo único, acumulação, exportação, summary

## Dependências
- `synthetic_university.thesis_enricher.ThesisEnricher`
- `synthetic_university.novelty_analysis.NoveltyAnalyzer`
- `synthetic_university.i18n` (para labels, opcional)

## Não Escopo
- R95 **não** implementa schedule real (cron/thread). O loop é acionado manualmente por `run_cycle()`.
- R95 **não** substitui o MCP server ou o pipeline batch existente.
